# Buffett x Dimon 토론: 고정 달러 리스크 → 순수 퍼센트(%) 기반 전환

**일시**: 2026-04-11  
**참석**: Warren Buffett (자본 보존 최우선), Jamie Dimon (기관 리스크 관리)  
**주제**: 실거래 리스크 파라미터를 고정 달러($100 상한/$10 하한) 대신 전부 퍼센트(%)로 통일할 것인가?  
**배경**: 사용자 자본 $200 USDT (Binance Futures), 현행 `risk = clamp(balance x 2%, $10, $100)`

---

## 문제 정의

현행 규칙 `clamp(balance x 2%, $10, $100)`의 규모별 왜곡:

| 자본 | 2% 계산값 | clamp 적용 후 | 실질 리스크% | 문제 |
|------|-----------|--------------|-------------|------|
| $200 | $4 | **$10** (하한) | **5.0%** | 2% 원칙 위반, 2.5배 초과 |
| $500 | $10 | $10 | 2.0% | 정상 |
| $2,000 | $40 | $40 | 2.0% | 정상 |
| $5,000 | $100 | $100 | 2.0% | 정상 (상한 도달 직전) |
| $10,000 | $200 | **$100** (상한) | **1.0%** | 절반으로 축소 |
| $50,000 | $1,000 | **$100** (상한) | **0.2%** | 극도로 보수적, 수익 기회 상실 |

---

## Round 1: 기본 입장

### Warren Buffett (조건부 찬성)

> "내가 4월 9일에 고정 달러 상한을 제안한 이유는 명확했네. 봇이 아직 검증되지 않았고, 절대 금액으로 손실 상한을 체감할 수 있게 하자는 것이었지."

- **인정하는 문제**: $200 계좌에서 MIN_RISK $10 적용 시 거래당 5% 리스크는 내 원칙(2%)에 정면으로 위배됨
- **$50,000 시나리오**: $100 상한이면 0.2%밖에 안 되어 시스템이 사실상 무력화. 하루 50거래를 해도 자본의 10%밖에 활용 불가
- **조건**: 퍼센트 전환은 좋지만, 반드시 **다층 방어선이 건재한지** 확인 필요. 퍼센트가 커지는 구간에서 안전장치가 작동하는지 검증해야 함

### Jamie Dimon (전적 찬성)

> "JP Morgan에서 신규 트레이더에게 고정 달러 리밋을 거는 건 수습 기간 3개월뿐이야. 이후엔 VaR 기반 퍼센트로 전환하지. 왜? 고정 달러는 스케일하지 않으니까."

- **기관 관행**: 모든 주요 기관은 % 기반 리스크 관리. 고정 달러는 온보딩용
- **핵심 논거**: 이미 일일 손실 티어(-3%, -5%), Peak Drawdown(-7%), Circuit Breaker(-20%)가 전부 % 기반. 거래당 리스크만 달러로 클램핑하는 건 **체계의 일관성을 깨뜨림**
- **자연 하한선**: Binance Futures 최소 노셔널 = $100. 자본이 너무 작아 노셔널 미달이면 거래 자체가 불가능 → 별도 MIN_RISK 불필요

---

## Round 2: 구체적 수치 공방

### 규모별 시뮬레이션 — 순수 2% 적용 시

#### $200 계좌

| 항목 | 현행 (clamp) | 제안 (순수 2%) |
|------|-------------|---------------|
| 거래당 리스크 | $10 (5%) | **$4 (2%)** |
| 포지션 사이즈 (3x, SL 1.5%) | $667 | $267 |
| 노셔널 $100 충족? | Yes | Yes ($267 > $100) |
| 일일 -3% 절반 트리거 | $6 손실 | $6 손실 |
| 일일 -5% 정지 트리거 | $10 손실 | $10 손실 |

**Buffett**: "$200에서 $4 리스크가 맞아. $10은 내 2% 원칙을 정면으로 위반하고 있었네. 소자본이라고 리스크 비율을 올리는 건 파산으로 가는 지름길이야."

**Dimon**: "노셔널 $267이면 Binance 최소($100)를 충족해. 다만 자본이 $150 이하로 떨어지면 2%=$3, 3x 레버리지로 노셔널 $200... 아직 충족. $50까지 가야 노셔널 미달이 발생하는데 그 전에 Circuit Breaker(-20%, $160→$128)가 작동하지."

#### $2,000 계좌

| 항목 | 현행 (clamp) | 제안 (순수 2%) |
|------|-------------|---------------|
| 거래당 리스크 | $40 (2%) | **$40 (2%)** |
| 포지션 사이즈 (4x, SL 1.2%) | $3,333 | $3,333 |
| 일일 -3% 절반 트리거 | $60 손실 | $60 손실 |
| 일일 -5% 정지 트리거 | $100 손실 | $100 손실 |

**양측 합의**: $500~$5,000 구간은 현행과 동일. 변화 없음.

#### $50,000 계좌

| 항목 | 현행 (clamp) | 제안 (순수 2%) |
|------|-------------|---------------|
| 거래당 리스크 | **$100** (0.2%) | **$1,000** (2%) |
| 포지션 사이즈 (5x, SL 1%) | $10,000 | **$100,000** |
| 일일 -3% 절반 트리거 | $1,500 손실 | $1,500 손실 |
| 일일 -5% 정지 트리거 | $2,500 손실 | $2,500 손실 |
| Peak Drawdown -7% | $3,500 | $3,500 |

**Buffett 우려**: "$50,000에서 거래당 $1,000은 상당한 금액이야. 3연패면 $3,000, 일일 -6%야."

**Dimon 반박**: 
> "Warren, 일일 -3%($1,500)에서 이미 사이즈 절반이야. 그러면 거래당 $500으로 줄어들지. -5%($2,500)에 도달하면 완전 정지. 3연패로 -6%가 되려면 **일일 티어를 모두 무시해야** 하는데, 그건 불가능해."

**Buffett 검증**:
> "맞아. 계산해보면... $1,000 손실(1패) → -2%, 2패째는 사이즈 절반이니 $500 → 누적 -3% → 절반... 사실상 3%를 넘기기 어렵고, 5%에 도달하려면 많은 연패가 필요하군. 일일 티어가 이미 방어하고 있어."

### 핵심 발견: 다중 방어선의 중복성

```
거래당 리스크 2% ← 1차 방어
       ↓ (2연패 = -4%)
일일 -3% 사이즈 절반 ← 2차 방어
       ↓ (절반 사이즈로 추가 손실)  
일일 -5% 당일 정지 ← 3차 방어
       ↓ (다음 날 리셋)
Peak Drawdown -7% ← 4차 방어
       ↓
Circuit Breaker -20% ← 5차(최종) 방어
```

**Dimon**: "5중 방어선에서 거래당 달러 클램프는 **1.5차 방어**에 불과해. 없어도 2차~5차가 건재하지."

**Buffett**: "인정하네. 달러 클램프를 빼도 방어 구조에 구멍이 생기지 않아."

## Round 3: 최종 합의

### 합의 결과 (만장일치)

#### 확정 파라미터

| 파라미터 | 이전 값 | **새 값** | 변경 사유 |
|---------|---------|----------|----------|
| 거래당 리스크 | `clamp(balance x 2%, $10, $100)` | **`balance x 2%`** | 클램프 제거, 순수 % |
| 최소 거래 조건 | MIN_RISK_DOLLARS = $10 | **노셔널 >= $100** | Binance 최소 요구사항으로 자연 하한 |
| Daily Loss Tier1 | -3% 사이즈 절반 | **-3% 사이즈 절반** | 유지 (CLAUDE.md 기준) |
| Daily Loss Tier2 | -5% 당일 정지 | **-5% 당일 정지** | 유지 (00시 복귀) |
| Peak Drawdown | -7% | **-7%** | 유지 |
| Circuit Breaker | -20% | **-20%** | 유지 |
| 슬리피지 버퍼 | 95% | **95%** | 유지 |
| 속도 제한 | 3연속 SL → 30분 | **3연속 SL → 30분** | 유지 |

#### 삭제 항목

| 삭제 파라미터 | 이전 값 | 삭제 사유 |
|-------------|---------|----------|
| ~~MAX_RISK_DOLLARS~~ | $100 | 순수 % + 일일 티어로 충분, 대자본에서 기회 상실 |
| ~~MIN_RISK_DOLLARS~~ | $10 | 노셔널 $100 자연 하한으로 대체, 소자본에서 2% 원칙 위반 유발 |

#### 코드 변경 필요사항

```python
# Before
risk = clamp(balance * 0.02, MIN_RISK_DOLLARS, MAX_RISK_DOLLARS)

# After
risk = balance * 0.02
notional = risk / sl_percent * leverage
if notional < 100:  # Binance minimum notional
    skip_trade("Notional below $100 minimum")
```

#### 설정값 변경

```
# 삭제
MAX_RISK_DOLLARS = 100       # 제거
MIN_RISK_DOLLARS = 10        # 제거

# 추가
MIN_NOTIONAL = 100           # Binance 최소 노셔널 ($100)

# 유지 (변경 없음)
```

## 핵심 인사이트

> **Buffett**: "고정 달러는 내가 봇을 신뢰하기 전에 걸어둔 훈련 바퀴였어. 다층 방어선이 검증된 이상, 훈련 바퀴는 오히려 자전거를 불안정하게 만들지."

> **Dimon**: "리스크 관리의 기본 원리는 일관성이야. 한 파라미터만 다른 단위를 쓰면 시스템이 예측 불가능해져. 전부 %면 어떤 규모에서든 동일한 행동을 보장하지."

## 결론

고정 달러 클램프(`MAX_RISK_DOLLARS`, `MIN_RISK_DOLLARS`) **완전 제거**.  
순수 퍼센트 체계(`balance x 2%`)로 전환.  
최소 거래 조건은 Binance 노셔널 $100으로 자연 적용.  
모든 자본 규모에서 동일한 2% 리스크, 동일한 방어 체계.
