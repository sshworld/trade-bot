# 워렌 버핏 x 제이미 다이먼 — 물타기 간격 vs SL 거리 최적화

**일시**: 2026-04-13
**안건**: 분할 진입(물타기) 간격이 SL 대비 지나치게 좁은 문제 해결
**현재 설정**: `entry_atr_offsets = [0.0, 0.5, 1.0]`, `entry_atr_offset_caps = [0.0, 1.0, 1.5]`

---

## 현황 분석

| 항목 | 값 | 비고 |
|------|-----|------|
| 30m ATR | ~$130 | |
| Entry 1 | $71,547 (0%) | 즉시 체결 |
| Entry 2 | $71,680 (+0.19%) | ATR x 0.5 = $65 |
| Entry 3 | $71,813 (+0.37%) | ATR x 1.0 = $130 |
| SL | $72,915 (+1.87%) | 잔고 2% 고정 손실 기준 |
| **Entry 간격 / SL 비율** | **0.37% / 1.87% = 20%** | 너무 좁음 |

**핵심 문제**: 마지막 Entry가 SL 거리의 20%에 불과 -> 3개 tranche가 거의 동시에 체결되어 물타기 효과가 없다.

---

## Round 1: 문제 정의와 프레임워크

### 워렌 버핏 (Warren Buffett)

물타기의 핵심은 "역행을 기회로 전환"하는 것입니다. 그런데 현재 간격(0.19%, 0.37%)은 BTC의 일반적인 1분 변동성 수준입니다. 즉, 시장가 주문 하나 넣는 사이에 3개가 다 체결됩니다.

제가 주식을 분할 매수할 때는 의미 있는 가격 역행이 있을 때만 추가합니다. SL까지의 거리를 3등분한다고 생각하면:

- **SL 거리 1.87%의 1/3 = 0.62%, 2/3 = 1.25%**

이 정도 간격이면 Entry 2는 "어느 정도 역행했을 때", Entry 3는 "꽤 역행했지만 아직 SL 전"에 체결됩니다. **옵션 C(SL 비례)**가 가장 원칙에 부합합니다.

다만 주의할 점: SL 거리는 잔고 2% 기반이므로 레버리지, 포지션 크기에 따라 변합니다. ATR과 무관하게 SL 거리에 연동하면 일관성이 유지됩니다.

**제안**: Entry 간격 = SL 거리의 **30% / 60%** (Entry 2 / Entry 3)

### 제이미 다이먼 (Jamie Dimon)

버핏의 분석에 동의하지만, 실무적 관점을 추가하겠습니다.

현재 시스템은 ATR 기반 offset에 % 캡을 씌우는 이중 구조인데, **ATR이 낮은 구간에서 간격이 과도하게 좁아지는 것**이 근본 원인입니다. 30m ATR $130은 BTC $71,500 대비 0.18%에 불과합니다.

SL 비례 방식의 문제점:
1. SL 거리는 포지션 생성 시점에 계산되는데, 물타기 주문은 사전에 배치해야 합니다
2. SL이 재계산되면 간격도 바뀌어야 하는데, 이미 배치된 주문을 수정해야 합니다

**실용적 대안**: ATR 배수를 올리되, **최소 % 하한(floor)을 추가**하는 것이 구현이 깔끔합니다.

| Tranche | ATR 배수 | 최소 % | 최대 % (캡) |
|---------|---------|--------|------------|
| Entry 1 | 0.0 | 0.0% | 0.0% |
| Entry 2 | 1.5 | **0.4%** | 1.0% |
| Entry 3 | 3.0 | **0.8%** | 1.5% |

30m ATR $130 기준: ATR x 1.5 = $195(0.27%) -> floor 0.4% 적용 -> $286
30m ATR $130 기준: ATR x 3.0 = $390(0.55%) -> floor 0.8% 적용 -> $572

**SL(1.87%) 대비 Entry 3 = 0.8% = 43%** -> 훨씬 효과적.

## Round 2: 구체적 수치 조율

### 워렌 버핏

다이먼의 floor 방식이 실용적이라는 점을 인정합니다. SL 비례 방식은 구현 복잡도가 높고, 이미 SL 사전 배치(STOP_MARKET) 구조에서 추가 연동은 위험합니다.

다만 수치를 조정하고 싶습니다:

**ATR 배수가 너무 공격적이면 안 됩니다.** ATR x 3.0이면 1h 타임프레임에서 ATR이 $400일 때 Entry 3가 $1,200(1.68%) 떨어지는데, 이때 SL이 1.87%라면 Entry 3 체결 직후 SL 발동할 수 있습니다.

제 수정 제안:

| Tranche | ATR 배수 | 최소 % (floor) | 최대 % (cap) |
|---------|---------|---------------|-------------|
| Entry 1 | 0.0 | 0.0% | 0.0% |
| Entry 2 | 1.0 | **0.3%** | 1.0% |
| Entry 3 | 2.0 | **0.7%** | 1.5% |

이렇게 하면:
- **저변동성** (ATR $130): floor 적용 -> 0.3% / 0.7% -> SL 대비 16% / 37%
- **중변동성** (ATR $300): ATR 적용 -> 0.42% / 0.84% -> SL 대비 22% / 45%
- **고변동성** (ATR $600): 캡 적용 -> 1.0% / 1.5% -> SL 대비 적정

Entry 3가 SL 거리의 **37~50%** 범위에 들어와 물타기 효과가 확보됩니다.

### 제이미 다이먼

버핏의 보수적 조정을 수용합니다. ATR x 3.0은 확실히 과했습니다. 다만 한 가지 더:

**Entry 비율도 재검토해야 합니다.** 현재 50/30/20인데, 간격을 넓히면 Entry 3 체결 확률이 낮아집니다. Entry 3에 20%만 배정하면 역행 시 평단 개선 효과가 미미합니다.

그러나 CLAUDE.md에 명시된 현재 합의(60/25/15, 2026-04-13 오프셋 전략 회의)를 존중해야 합니다. 비율 변경은 별도 안건으로 다루겠습니다.

**구현 관점 정리**:
1. `entry_atr_offsets` = [0.0, 1.0, 2.0] (배수 상향)
2. `entry_atr_offset_caps` = [0.0, 1.0, 1.5] (캡 유지)
3. **신규 파라미터** `entry_atr_offset_floors` = [0.0, 0.3, 0.7] (최소 % 하한 추가)

`_create_entry_tranches`에서 offset 계산 후 floor를 적용하는 로직:
```
offset = max(atr * multiplier, base_price * floor_pct / 100)
offset = min(offset, base_price * cap_pct / 100)
```

## Round 3: 최종 합의와 엣지 케이스

### 워렌 버핏

최종 확인 사항:

1. **소자본 자동 분할 조정과의 호환성**: 잔고가 적어 2분할/1분할로 줄어들 때, floor가 어떻게 작동하는가?
   - 2분할이면 Entry 2만 사용 -> floor 0.3% 적용, 문제 없음
   - 1분할이면 floor 무관, 문제 없음

2. **SHORT 포지션**: SHORT 물타기는 가격 상승 시 추가 매도. SL은 아래에 있으므로 방향만 반대, 비율은 동일하게 적용. 문제 없음.

3. **SL 재계산 시**: 물타기 체결 후 평단이 바뀌면 SL도 재계산됩니다. 이때 Entry 간격은 이미 배치된 주문이므로 영향 없음. 다만 SL이 좁아져서 나머지 Entry가 SL 바깥에 놓일 수 있는데, 이는 오히려 **좋은 것**입니다 -- 과도한 물타기를 방지합니다.

**최종 동의**: [0.0, 1.0, 2.0] + floor [0.0, 0.3, 0.7] + cap [0.0, 1.0, 1.5]

### 제이미 다이먼

모든 엣지 케이스를 검토했고, 동의합니다.

**시뮬레이션 비교 (BTC $71,500, 30m ATR $130, SL 1.87%)**:

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| Entry 2 offset | 0.19% ($65) | **0.30% ($215)** |
| Entry 3 offset | 0.37% ($130) | **0.70% ($500)** |
| Entry 간격 / SL | 20% | **37%** |
| Entry 3 체결 조건 | 가격 0.37% 역행 | 가격 0.70% 역행 |
| 물타기 평단 개선 | ~0.1% | **~0.3%** |

**고변동성 시나리오 (ATR $600)**:

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| Entry 2 offset | 0.42% ($300) | **0.84% ($600)** |
| Entry 3 offset | 0.84% ($600) | **1.50% ($1,073)** cap |
| 물타기 효과 | 중간 | **우수** |

실무적으로 `schemas.py`에 `entry_atr_offset_floors` 필드 추가, `engine.py`와 `live_engine.py`의 `_create_entry_tranches`에 floor 로직 3줄 추가면 됩니다.

### 로직 변경

```python
# _create_entry_tranches 내 offset 계산
offset_price = Decimal(str(atr * atr_mult))
# 최소 % 하한 (floor) 적용
if floor_pct > 0:
    min_offset = base_price * Decimal(str(floor_pct / 100))
    offset_price = max(offset_price, min_offset)
# 최대 % 캡 적용
if cap_pct > 0:
    max_offset = base_price * Decimal(str(cap_pct / 100))
    offset_price = min(offset_price, max_offset)
```
