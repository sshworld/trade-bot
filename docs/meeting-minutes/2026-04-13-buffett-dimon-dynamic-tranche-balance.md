# 전문가 토론: 동적 분할 진입/청산 + 잔고 표시 최적화

**일시**: 2026-04-13
**참석**: Warren Buffett (리스크 관리), Jamie Dimon (실행/수익 최적화)
**안건**: $200 소자본 환경에서의 진입/청산 분할 동적화, 리스크 수준, 잔고 표시 체계

---

## 배경

- 잔고: $200, 레버리지 5x, 리스크 2%
- Binance 제약: 최소 0.001 BTC 단위, 최소 $100 노셔널
- BTC 가격 ~$84,000 기준
- 동시 포지션: 1개만
- Cross margin 모드

### 현재 문제점
1. 분할 비율이 하드코딩 (3분할 50/30/20, 2분할 60/40)
2. $200에서 3분할 시 일부 tranche가 최소 노셔널 미달
3. 마진 표시가 cross margin과 맞지 않음 (로컬 차감 혼란)
4. 자본 성장에 따른 자동 스케일업 없음

---

## Buffett 입장

- $200에서 무리한 3분할보다 **수학에 맞는 동적 분할**
- 리스크 인상에 신중하나, 안전장치 존재하므로 2.5% 수용 가능
- 레버리지 5x 절대 유지 (4/10 확정 사항)
- 최저 운영 잔고 $100 설정 필요

## Dimon 입장

- ATR 기반 동적 분할이 하드코딩보다 우월
- 2.5% 리스크로 SL 여유 확보 (단일 포지션이라 안전)
- 청산 분할 수 = 진입 분할 수 연동
- 1분할 시 TP1 전량 청산 (0.002 BTC 미분할)

## 최종 합의

| 항목 | 현행 | 변경 | 근거 |
|------|------|------|------|
| **리스크/거래** | 2% | **2.5%** | 단일 포지션 + 안전장치 존재. SL 여유 확보 |
| **레버리지** | 5x | **5x 유지** | 4/10 확정사항. 크로스마진 청산리스크 |
| **분할 진입 수** | 고정 3/2 | **동적 (ATR 기반)** | floor(effective_size / 0.002), max는 시그널 타입별 |
| **WITH_TREND max** | 3 | **3 유지** | 자본 커지면 자동 활성화 |
| **COUNTER max** | 2 | **2 유지** | 변경 불필요 |
| **분할 청산 수** | 고정 3 | **진입 분할 수 연동** | 각 exit tranche >= 0.002 BTC 보장 |
| **1분할 시 TP** | 3단계 | **TP1 전량 청산** | 0.002 BTC 미분할, 확정수익 우선 |
| **잔고 표시** | 로컬 마진 차감 | **Binance 실잔고** | 크로스마진 정확성 |
| **최저 운영 잔고** | 없음 | **$100** | 이하 시 자동 거래 중단 |
| **TP1→BE** | 현행 유지 | **현행 유지** | 수수료 포함 브레이크이븐 정확 |

### 동적 분할 계산 로직

```
sl_distance = atr * tf_sl_multiplier
position_size = (balance * 0.025) / sl_distance
effective_size = position_size * 0.95  # slippage buffer

# Entry tranches
max_tranches = 3 if WITH_TREND else 2
num_entry = min(floor(effective_size / min_tranche_qty), max_tranches)
num_entry = max(num_entry, 1)

# Exit tranches = 진입 분할 수 연동
num_exit = num_entry
# tail merge로 min_tranche 미달분 합침
```
