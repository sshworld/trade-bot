# Buffett x Dimon: 분할 진입 방향 — 피라미딩 vs 평단 최적화
## 2026-04-11

---

## 배경

현재 WITH_TREND 분할 진입은 **피라미딩** 방식:
- LONG: `base * (1 + offset)` → 가격 상승 시 추가 매수 (평단 불리)
- SHORT: `base * (1 - offset)` → 가격 하락 시 추가 매도 (평단 불리)
- offsets: [0%, +0.3%, +0.6%], split: [50%, 30%, 20%]

사용자 질문: "분할 매수 자체가 평단가 낮춰서 안정성을 유지하는 것이 아닌가?"

제안: 방향 반전하여 **역행 시 추가** (평단 유리):
- LONG: 가격 하락 시 추가 매수 → 평단 낮아짐
- SHORT: 가격 상승 시 추가 매도 → 평단 높아짐

참고: COUNTER_TREND은 이미 [0%, -0.3%]로 역행 추가(평단 유리) 방식.

---

## Round 1: 기본 입장 — 전통적 분할매수 vs 피라미딩

### Warren Buffett (자본 보존)

전통적 분할매수(DCA, 물타기)의 핵심은 **평균 단가 최적화**다. 내가 코카콜라를 살 때도 가격이 내려오면 더 샀다. 이유는 단순하다:

1. **평단가가 낮아지면 손익분기점이 가까워진다.** 같은 반등으로도 수익 전환이 빠르다.
2. **50% 즉시 체결이라는 점이 중요하다.** 첫 진입은 이미 시장가로 들어간다. 나머지 50%는 "더 좋은 가격에 사겠다"는 것이 합리적이다.
3. **리스크 관점:** 가격이 역행하면 더 좋은 가격에 추가 → 평단 개선. 가격이 즉시 순방향이면 50%만 탄다 — 이것도 나쁘지 않다.

현재 피라미딩 방식은 **"이미 이기고 있을 때만 추가"** 라는 철학인데, $200 소자본 단타에서 +0.3%, +0.6% 올라간 후 추가 매수하면 평단만 올라가고, 반전 시 손실이 가중된다.

### Jamie Dimon (기관 리스크)

피라미딩에도 논리가 있다. **모멘텀 확인(confirmation)** 이라는 관점이다:

1. **가격이 +0.3% 더 올라갔다 = 방향 확인.** 30% 추가 투입의 근거가 생긴다.
2. **전통 트렌드 추종 전략에서 피라미딩은 표준이다.** Turtle Traders가 대표적이다.
3. **역행 추가(물타기)의 위험:** 방향이 완전히 틀리면 풀 사이즈가 손실 구간에 진입한다.

그러나... 솔직히 말하면, 이 시스템 맥락에서는 의문이 있다:
- offset이 **0.3%, 0.6%** — 극히 작은 범위다. "확인"이라고 부르기엔 너무 작다.
- BTC/USDT에서 0.3%는 약 $280 (BTC=$93,000 기준). 노이즈 수준이다.
- **진짜 모멘텀 확인이라면 1-2% 이상 움직여야 의미가 있다.**

## Round 2: 레버리지 5x 선물 특수성

### Warren Buffett

여기서 결정적인 차이가 나온다. **선물은 현물이 아니다.**

| 구분 | 현물 | 5x 선물 |
|------|------|---------|
| 청산 리스크 | 없음 | 있음 (마진 소진 시) |
| 물타기 위험 | 낮음 (최악에 존버) | 높음 (청산 가능) |
| 피라미딩 장점 | 보통 | 높음 (SL이 자동 축소) |

**그러나** 이 시스템은 SL이 ATR 기반으로 걸려있고, 리스크 2%로 사이징한다. 청산까지 가려면 SL이 작동하지 않아야 한다. SL이 정상 작동하는 한, **물타기든 피라미딩이든 최대 손실은 동일하게 2%다.**

핵심 질문: **SL이 동일하다면, 평단이 유리한 쪽이 더 자주 승리하지 않는가?**

### Jamie Dimon

맞다. 리스크 관리 프레임워크를 보자:

**현재 안전장치:**
- SL: 1.2-2.0 x ATR (TF별)
- 일일 손실 -3% → 사이즈 절반
- 일일 손실 -5% → 당일 중단
- Drawdown -7% → 당일 중단 (Live)
- 3연속 SL → 30분 중단

이 모든 장치가 **최대 손실을 제한**한다. 그렇다면 진입 방식은 **승률(win rate)과 평균 수익(avg win)** 에 영향을 준다.

피라미딩 시나리오 (현재):
- 방향 맞음: 풀 사이즈 진입, 평단 불리하지만 수익 구간
- 방향 틀림: 풀 사이즈 진입, 평단 불리하므로 SL 더 빨리 도달

역행 추가 시나리오 (제안):
- 방향 맞음: 50%만 탄 채 수익 (나머지 미체결)
- 방향 틀림: 풀 사이즈, 평단 유리하므로 SL까지 여유

**선물 5x에서 평단이 유리하면 SL까지의 거리가 실질적으로 넓어진다.** 같은 SL 가격이라도 평단이 낮으면(LONG 기준) 실현 손실이 줄어든다.

## Round 3: 수치 시뮬레이션

### 시나리오 설정
- 자본: $200, 레버리지: 5x, 리스크: 2% ($4)
- BTC 진입가: $93,000, SL: 1.5 x ATR (ATR=$400, SL거리=$600)
- SL가격(LONG): $92,400
- 포지션 노셔널: $4 / (600/93000) = $620 → qty = 0.00667 BTC
- 슬리피지 95%: qty = 0.00634 BTC

### Case A: 피라미딩 (현재) — LONG

| Tranche | 체결 조건 | 가격 | 수량 | 비용 |
|---------|----------|------|------|------|
| T1 (50%) | 즉시 | $93,000 | 0.00317 | $294.81 |
| T2 (30%) | $93,279 (+0.3%) | 0.00190 | $177.23 |
| T3 (20%) | $93,558 (+0.6%) | 0.00127 | $118.82 |

**풀 체결 시 평단: $93,186** (가중평균)
SL $92,400까지 거리: $786 → 실현 손실: 0.00634 x $786 = **$4.98** (리스크 2.49%)

### Case B: 역행 추가 (제안) — LONG

| Tranche | 체결 조건 | 가격 | 수량 | 비용 |
|---------|----------|------|------|------|
| T1 (50%) | 즉시 | $93,000 | 0.00317 | $294.81 |
| T2 (30%) | $92,721 (-0.3%) | 0.00190 | $176.17 |
| T3 (20%) | $92,442 (-0.6%) | 0.00127 | $117.40 |

**풀 체결 시 평단: $92,814** (가중평균)
SL $92,400까지 거리: $414 → 실현 손실: 0.00634 x $414 = **$2.62** (리스크 1.31%)

### Warren Buffett

수치가 명확하다:

| 시나리오 | 평단(LONG) | SL까지 거리 | 풀체결 시 최대손실 |
|---------|-----------|------------|------------------|
| 피라미딩 | $93,186 | $786 | $4.98 (2.49%) |
| 역행추가 | $92,814 | $414 | $2.62 (1.31%) |

**피라미딩이 풀 체결되면 리스크가 2%를 초과한다!** 평단이 올라가서 SL까지의 실현 손실이 커지기 때문이다.

반면 역행 추가는 풀 체결 시 리스크가 1.31%로 **줄어든다.** 평단이 SL에 가까워지지만, 진입 가격이 낮아서 손실 단가가 줄어든다.

### Jamie Dimon

잠깐. **방향이 맞을 때를 비교하자:**

방향 맞음 — 가격이 $93,500까지 오른 경우:

| 시나리오 | 체결 수량 | 평단 | 수익 |
|---------|----------|------|------|
| 피라미딩 | 0.00634 (풀) | $93,186 | 0.00634 x $314 = $1.99 |
| 역행추가 | 0.00317 (50%만) | $93,000 | 0.00317 x $500 = $1.59 |

피라미딩이 **25% 더 수익**이다. 풀 사이즈가 수익 구간에 있으니까.

그런데... $93,500까지 간 후 $92,721까지 빠졌다가 다시 올라가는 경우:
- 역행 추가: T2 체결 → 80% 진입 → 반등 시 더 많은 수량으로 수익
- 피라미딩: 이미 풀 체결 상태 → 결과 동일

**핵심 트레이드오프:**
- 피라미딩: 승리 시 수익↑, 패배 시 손실↑
- 역행 추가: 승리 시 수익↓(미체결 존재), 패배 시 손실↓(평단 유리)

## Round 4: 기존 안전장치와의 조합

### Warren Buffett

시스템 전체를 보자:

1. **SL 기반 사이징 (2%)**: 이것이 1차 방어선. 그런데 피라미딩 시 **풀 체결되면 실현 리스크가 2.49%로 올라간다** — 설계 의도를 벗어난다.

2. **일일 손실 티어**: -3%에서 사이즈 반감. 피라미딩으로 거래당 2.49% 손실이면 2번만 지면 -4.98% → 즉시 정지. 역행 추가라면 2번 져도 -2.62% → 아직 Tier 1에도 안 걸린다.

3. **3연속 SL 중단**: 피라미딩에서 3연패 = -7.47%. 역행 추가에서 3연패 = -3.93%. **생존 마진이 2배 차이.**

4. **COUNTER_TREND은 이미 역행 추가 [0%, -0.3%]**: 설계자가 counter에는 물타기가 맞다고 판단한 것. 그렇다면 WITH_TREND도 동일 논리 적용이 자연스럽다.

### Jamie Dimon

기관 관점에서 핵심 포인트:

**현재 피라미딩의 설계 의도는 "confirmation"이었다.** 가격이 +0.3% 더 가면 방향 확인으로 추가. 그러나:

1. **0.3%는 confirmation이 아니다.** BTC에서 $280은 1분 노이즈. 진짜 confirmation이라면 ATR의 0.5배 이상이어야 한다.
2. **offset이 너무 작아서 피라미딩의 이점(모멘텀 확인)을 누리지 못한다.** 노이즈에 불과한 움직임으로 풀 사이즈 진입하게 되므로, 사실상 "비싸게 사는" 효과만 있다.
3. **역행 추가의 offset도 0.3%로 작다.** 하지만 작은 역행에 추가 매수하는 것은 "약간 할인된 가격에 사는" 효과 → 이쪽이 합리적.

**SL 재계산 로직 확인:**
코드를 보면 `_recalculate_position`에서 tranche 체결 시 평단과 SL을 재계산한다. 역행 추가로 전환하면:
- 평단이 낮아지고 → SL도 재계산 → **SL이 더 낮아져서 여유 확보**
- 피라미딩에서는 평단이 높아지고 → SL도 재계산 → **SL이 올라가서 촉발 확률 증가**

## Round 5: 최종 합의

### Warren Buffett

내 결론은 명확하다. **역행 추가(평단 최적화)로 전환해야 한다.**

근거:
1. 풀 체결 시 리스크가 2% 이내로 유지됨 (피라미딩은 2.49%로 초과)
2. 안전장치와의 시너지가 우수 (일일 손실 티어에 더 늦게 도달)
3. 0.3% offset은 모멘텀 확인으로 의미 없음 — 할인 매수로는 의미 있음
4. COUNTER_TREND과 일관성 확보
5. $200 소자본에서 생존이 최우선

### Jamie Dimon

동의한다. 단, **조건부 동의**:

1. **offset 값은 유지 [0%, -0.3%, -0.6%]**: 너무 크게 벌리면 미체결 리스크 증가
2. **SL과 T3의 간격 확인 필요**: LONG에서 T3가 -0.6%($92,442)이고 SL이 $92,400이면 불과 $42 차이 → SL이 먼저 걸릴 수 있음
3. **`_should_fill_entry` 로직은 이미 호환**: LONG은 `price <= target_price`로 체결하므로, target이 base보다 낮으면 자연스럽게 역행 시 체결된다.

### 공동 합의

**WITH_TREND 진입 방향을 역행 추가(평단 최적화)로 전환한다.**

구체적 변경:

```python
# AS-IS (피라미딩)
entry_offsets: list[float] = [0.0, 0.3, 0.6]
# → LONG: base * (1 + 0.003), (1 + 0.006)  ← 높게 추가매수
# → SHORT: base * (1 - 0.003), (1 - 0.006)  ← 낮게 추가매도

# TO-BE (평단 최적화)
entry_offsets: list[float] = [0.0, -0.3, -0.6]
# → LONG: base * (1 - 0.003), (1 - 0.006)  ← 낮게 추가매수 (평단↓)
# → SHORT: base * (1 + 0.003), (1 + 0.006)  ← 높게 추가매도 (평단↑)
```

**코드 변경은 `entry_offsets` 부호만 바꾸면 된다.** 엔진 로직(`_create_entry_tranches`)은 offset 부호에 따라 자동으로 방향이 결정되므로 수정 불필요.

변경 파일: `backend/app/trading/schemas.py`
```python
# TradingSettings (line ~170)
entry_offsets: list[float] = [0.0, -0.3, -0.6]   # 역행 추가 (평단 유리)

# CounterTrendSettings (line ~151) — 변경 없음 (이미 역행)
entry_offsets: list[float] = [0.0, -0.3]
```

주석 변경: `"확인 추가"` → `"평단 최적화 (역행 시 추가)"`

### 주의사항
1. **SL과 T3 간격 모니터링**: T3(-0.6%)가 SL 근처면 T3가 미체결 상태로 SL 촉발 → 이 경우 50-80%만 진입된 채 청산되므로 손실이 더 작아짐 (오히려 장점)
2. **방향이 맞을 때 미체결 리스크**: 가격이 바로 순방향이면 50%만 탄다. 이는 수익 감소이지만, 안정성과의 트레이드오프로 수용
3. **COUNTER_TREND과 동일 방향이 됨**: 둘 다 역행 추가이며 offset 크기만 다름 — 이것이 오히려 일관성 있는 설계

## 결론

| 항목 | 결정 |
|------|------|
| WITH_TREND entry_offsets | `[0.0, 0.3, 0.6]` → **`[0.0, -0.3, -0.6]`** |
| 엔진 로직 변경 | **불필요** (offset 부호가 방향 결정) |
| COUNTER_TREND | **변경 없음** (이미 `[0.0, -0.3]`) |
| 주석 업데이트 | "확인 추가" → "평단 최적화" |
| 리스크 영향 | 풀 체결 시 2.49% → 1.31% (안전) |

**Buffett 서명: "Rule No.1: Never lose money. 평단을 유리하게."**
**Dimon 서명: "offset 부호 하나의 차이가 연간 수십 번의 거래에서 복리로 작용한다."**
